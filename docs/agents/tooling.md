# Tooling

## Builds and testing

- A **developer-supplied APK** is available for manual checks when you cannot or should not block on a local build.

## Android Auto

- There is **no** Android Auto–specific MCP in this workspace by default. Use **Android Studio**, Google’s **Android for Cars** documentation, the **Desktop Head Unit (DHU)**, and real hardware when possible.

## Other MCP integrations

- Depending on Cursor setup, you may see servers such as Convex, Figma, Hugging Face, Svelte, or tldraw. Use them when they fit the task.

## Figma and mockups

- Mockups should follow **approved user stories** and **PA2Theme** on phone surfaces.
- Car (Android Auto) previews should read as **host media** — neutral chrome, content from the session. Frame labels: [prototype-and-engineering.md](prototype-and-engineering.md).
- **Branch:** keep Figma notes and links under `**mockups`** (e.g. `mockups/research/`).
